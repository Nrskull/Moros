### 2026-04-09 / 聊天室紧凑化与头像预览

- 需求概述：压缩聊天室消息区与输入区的冗余留白；连续消息按同一角色分组；在编辑角色卡弹层中增加头像上传入口和即时预览。
- 实现结果：聊天气泡改为内容自适应宽度并收紧内边距；连续消息会隐藏重复的名字和时间戳，并缩小组内间距；角色切换按钮和输入框收纳进同一个紧凑底栏容器；编辑角色卡弹层新增圆形头像上传区，支持点击触发隐藏文件输入和本地预览。
- 关联文件：`src/lib/ChatRoomPage.svelte`、`src/app.css`、`doc/new需求更新.md`

### 2026-04-09 / 聊天室角色快照与头像持久化

- 需求概述：聊天消息显示角色名而非聊天室昵称；角色头像需要持久化并在聊天中展示；消息必须保存发送当时的角色快照，后续切换或修改角色不能回改历史消息；同时增大聊天气泡与聊天容器左右边界的距离。
- 实现结果：聊天室消息新增 `speakerName / speakerAvatarDataUrl / speakerCharacterId` 快照字段，普通消息和骰子结果都会在发送时固化当前角色身份；角色卡新增头像数据持久化，创建和编辑角色卡都可上传头像；聊天流渲染改为优先显示角色名与头像，连续消息分组也基于角色快照而不是昵称；消息列表左右留白同步增大。
- 关联文件：`server/chat-server.js`、`src/lib/chat-room.ts`、`src/lib/ChatRoomPage.svelte`、`src/app.css`、`doc/new需求更新.md`

### 2026-04-09 / 聊天室固定用户与无昵称登录

- 需求概述：为聊天室预置独立用户和密钥，不再让所有人共用 `admin` 身份；删除“进入聊天室前输入昵称”的步骤，改成密钥登录后直接匹配用户身份、角色卡和可进入房间。
- 实现结果：新增固定种子用户 `edie / lateraven / sakin / chen`，首次启动会自动把它们写入数据库并生成独立密钥；种子用户数据和密钥在后续重启中会保持不变，除非显式修改代码或数据库；聊天室连接流程删除昵称输入，改成密钥登录后直接以用户身份进入；房间列表和进房校验开始按用户权限过滤，普通用户只会看到自己有权限的房间，自建房默认只授权创建者进入。
- 关联文件：`server/chat-server.js`、`src/lib/ChatRoomPage.svelte`、`doc/Archive/需求更新存档.md`
