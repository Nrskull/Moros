# log编辑页需求


## 需求陈列（已经实现的可以注释掉）

## log上传编辑页

- [已完成] 右侧角色颜色选取的按钮和文字相似大小，角色管理手风琴折叠
    颜色选取按钮只需要大约20px的长宽，不应使用圆形包裹
- [已完成] 新增：每一条的最右边有可以点选的按钮，用于控制是否保留，正文默认勾选，符合场外发言规则的默认不勾选，CQ图片默认不勾选
    保留状态不显示说明文字，仅显示勾选框
- 新增和角色管理同级的“显示设置”，和角色管理同样，使用手风琴折叠，包含功能：仅显示正文、每页显示多少条、跳转至多少页。打开页面时角色管理和显示设置都默认为折叠状态。
## 当前实现备注

- 日志模块当前已拆成 `log展示页 -> log上传编辑页 / log播放页` 三段结构，编辑页只负责当前日志的修改与导出。
- 当前默认使用 `public/logs/test/sealdice-standard-log.json` 作为本地测试示例，并可从展示页的“新增上传 log”入口继续上传新的 `.json` 文件。
- 已支持按页编辑日志条目，当前主编辑区保留角色名与正文的高密度编辑。
- 已支持按角色自动分配默认颜色，并允许用户在右侧颜色映射面板中手动调整。
- 已支持实时生成修改版 JSON，并通过“下载修改版”按钮导出。
- 编辑页已删除时间修改入口，不再显示条目序号、骰点标签和记录时间，单条日志只保留角色名与正文。
- 编辑页已删除开发期摘要卡片、JSON 预览和 `log上传编辑页` 的大标题，只保留小灰字形式的“当前日志：日志名”。
- 点击分页栏的上一页 / 下一页后，视野会平滑滚回编辑区顶部。
- 右侧角色管理区已改成手风琴折叠；角色颜色按钮尺寸已压到接近文字输入框高度。
- 右侧角色管理已按角色身份键而不是当前显示名分组，即使用户把两个角色改成同名，也不会被系统自动合并。
- 每条日志最右侧新增“是否保留”勾选；正文默认保留，整条括号包裹的场外发言和 `CQ:image` 图片消息默认不保留。
- 播放页已改成主容器宽度的 `16:9` 播放块，正文固定落在底部四分之一高度区域，并按逐字方式播放。
- 已播放内容不再常驻显示，改为“回顾”按钮按需展开最近 10 条，并在展开时自动暂停播放。
- 当前日志需求后续只读取 `doc/log展示页需求.md` 与 `doc/log编辑页需求.md`，`Archive` 目录只作为历史留存，不再作为当前需求来源。
